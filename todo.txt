To display output from multiple subprocesses in segregated areas of the terminal, you need a library that can create a Terminal User Interface (TUI). This is exactly what your todo.txt item about a "dual-pane terminal dashboard" describes.

In the Go ecosystem, the idiomatic and most powerful library for this is Bubble Tea (charmbracelet/bubbletea).

How it works: Instead of just printing lines, a Bubble Tea application takes full control of the terminal screen. It operates on a Model-View-Update architecture:

Model: A struct that holds the entire state of your application (e.g., a list of recent log messages, a map of active downloads with their latest output lines).
Update: A function that processes incoming "messages" (like a new log line, a line of output from a subprocess, a keypress) and updates the Model accordingly.
View: A function that takes the current Model and returns a string representing the entire screen layout. Bubble Tea then efficiently draws this to the terminal.
To achieve your goal, you would pipe the stdout of each twitch-dlp or yt-dlp subprocess. As new lines of output arrive, you would send them as messages to your Bubble Tea Update function. The Update function would store these lines in the Model, and the View function would render them in the correct pane on the screen.

This approach allows you to build complex, interactive, and real-time dashboards directly in the terminal, perfectly solving the problem of monitoring multiple concurrent processes.

Ideal terminal:
Horizontal split. The left pane (70% width) should display a scrollable history of main orchestrator logs (e.g., initial text, status checks, 'channel found', 'process started'). The right pane (30% width) must be a dynamic grid that automatically splits vertically to accommodate one sub-pane for every active yt-dlp/twitch-dlp process. Each sub-pane should display the real-time stdout (verbose output and progress bar) of its respective yt-dlp/twitch-dlp instance. Ensure the script handles non-blocking subprocess reading to prevent the UI from freezing, and correctly cleans up child processes upon exit. The solution must be cross-platform compatible for Windows and Linux.
IMPORTANT: Each split pane should be scrollable and line wrap to ensure NOTHING IS LOST. The user should also be able to click and drag any text to copy as per usual terminal behavior.
